OAuth Scope Disclosure

LocalApex requests the https://www.googleapis.com/auth/business.manage scope. Here is exactly what it is used for.

Why LocalApex needs business.manage

This is Google's standard OAuth scope for all GBP APIs. LocalApex uses it only for the features explicitly listed below, all triggered by the authenticated user.

📊
Read search performance data
Fetches impressions, clicks, direction requests, and call counts to display analytics in the LocalApex dashboard.
⭐
Read and reply to customer reviews
Lists all Google reviews and posts owner replies on behalf of the authenticated business account — only when the user explicitly triggers the action.
🏢
Read and update business information
Retrieves and updates business profile fields (name, address, phone, hours, categories) that users explicitly edit within LocalApex.
👤
List accounts and locations
Enumerates GBP accounts and linked locations so users can select which listing to manage.
🔔
Read and configure notification settings
Reads and optionally updates which email addresses receive GBP alerts (new reviews, Q&A) as configured by the user.
📅
Manage Place Action (booking/CTA) links
Creates and updates booking and reservation URLs that appear as call-to-action buttons on the Google listing.
✅
Check verification status
Reads whether a location is verified on Google so the dashboard can surface an alert if action is needed.
🏨
Read and update lodging attributes
For hospitality businesses, reads and updates amenities and room attributes that users configure in the profile editor.

Data Commitment

LocalApex handles Google Business Profile data in strict accordance with the Google API Services User Data Policy and Limited Use requirements.

🔒

No data sold or shared

GBP data is never sold, rented, or shared with third parties. It is used solely to power the features you see in your LocalApex dashboard.

🎯

Minimal scope usage

Only the specific API endpoints required for each user-triggered action are called. No speculative or background data collection occurs.

🗑️

Data deletion on request

Users can disconnect Google and request full deletion of all stored GBP data at any time from the Settings page — honoured within 30 days.

🔑

Secure OAuth 2.0 only

Google passwords are never stored. Authentication uses OAuth 2.0 refresh tokens, revocable at any time from the user's Google account.

👁️

Transparent data access

Every API read and write is logged. Users know exactly when LocalApex accesses their Google Business Profile data.

📋

Limited Use compliant

All Google data use complies with the Limited Use requirements. Data is used only to provide features explicitly described on this page.
